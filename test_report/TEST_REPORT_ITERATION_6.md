## 发现的问题

### 🔴 问题 1: 详细健康检查数据库错误

**问题描述**：
- `/api/monitor/health/detailed` 返回 database 状态为 "unhealthy"
- 错误信息: "Textual SQL expression 'SELECT 1' should be explicitly declared as text('SELECT 1')"

**影响**：健康检查功能不准确

**修复建议**：修复数据库连接检查的SQL语法

### ⚠️ 问题 2: 错误响应状态码

**问题描述**：
- 无效股票代码返回 HTTP 200，但实际是业务错误
- 应返回 422 或 400

**影响**：客户端无法根据HTTP状态码判断请求是否成功

**修复建议**：业务错误返回适当的4xx状态码
