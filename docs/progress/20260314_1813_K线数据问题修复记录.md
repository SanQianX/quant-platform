## 时间

## 问题描述

### 问题1：K线数据与实际不匹配
- **现象**：前端显示的茅台(600519)价格约1700元，但实际应该是1413元
- **原因**：
  1. docker-compose.yml 中 ENV=development 导致后端连接 SQLite（只有25条旧数据），而不是 PostgreSQL
  2. 数据库查询返回的数据排序有问题

### 问题2：环境配置错误
- **现象**：后端使用了 SQLite 数据库而不是 PostgreSQL
- **原因**：docker-compose.yml 中 ENV 设置为 development

## 修复方案

### 修复1：修改环境配置
文件：`F:\quant-platform\deploy\docker-compose.yml`

```yaml
# 修改前
environment:
  - ENV=development

# 修改后
environment:
  - ENV=production
```

### 修复2：修改数据查询逻辑
文件：`F:\quant-platform\backend\services\stock_service.py`

- 确保查询使用 `limit(500)` 获取足够数据
- 确保数据按日期升序排列（最旧→最新）

### 修复3：修复数据库模型列名
文件：`F:\quant-platform\backend\models\stock.py`

- 表名：`kline_daily`（不是 `klines`）
- 列名：`date` → `trade_date`
- 列名：`volume` → `vol`

## 注意事项

1. **环境变量很重要**：
   - `ENV=development` → 连接 SQLite（本地开发）
   - `ENV=production` → 连接 PostgreSQL（生产环境）

2. **数据库模型与实际表结构必须一致**：
   - 表名：`kline_daily`
   - 列名：`trade_date`, `vol` 等

3. **数据排序**：
   - 前端K线图通常期望升序数据（最旧→最新）

4. **缓存问题**：
   - 重启容器可以清除缓存
   - 或者修改代码后重新构建镜像
